# 指标定义与计算规则

## 规模指标（用户侧）

### 日活 (DAU)
- **定义**：当日至少完成一次核心操作的去重用户数
- **计算**：COUNT(DISTINCT user_id) WHERE date = 当日 AND event_type IN ('upload', 'download', 'login')
- **更新频率**：每日凌晨3点计算前一日数据
- **正常波动范围**：±15%（工作日vs周末）

### 月活 (MAU)
- **定义**：当月至少完成一次核心操作的去重用户数
- **计算**：COUNT(DISTINCT user_id) WHERE date IN (当月) AND event_type IN ('upload', 'download', 'login')
- **更新频率**：每日更新（滚动30天）
- **正常波动范围**：±5%（月度环比）

### 上传用户数
- **定义**：当日完成上传操作的去重用户数
- **计算**：COUNT(DISTINCT user_id) WHERE date = 当日 AND event_type = 'upload'
- **关联指标**：上传容量、人均上传量

### 下载用户数
- **定义**：当日完成下载操作的去重用户数
- **计算**：COUNT(DISTINCT user_id) WHERE date = 当日 AND event_type = 'download'
- **关联指标**：下载容量、人均下载量

## 价值指标（资源侧）

### 上传总容量
- **定义**：当日所有用户上传文件的总容量
- **计算**：SUM(file_size) WHERE date = 当日 AND event_type = 'upload'
- **单位**：GB（原始字节转GB，保留2位小数）
- **正常波动范围**：±20%（受大文件影响）

### 下载总容量
- **定义**：当日所有用户下载文件的总容量
- **计算**：SUM(file_size) WHERE date = 当日 AND event_type = 'download'
- **单位**：GB
- **正常波动范围**：±25%（受大文件影响）

## 衍生关键比率

### 人均容量
- **定义**：总容量 / 活跃用户数
- **计算**：upload_capacity / DAU
- **意义**：判断用户是"浅层使用"还是"深度依赖"
- **健康值参考**：
  - 人均上传容量 > 100MB：深度用户
  - 人均上传容量 10-100MB：中度用户
  - 人均上传容量 < 10MB：浅层用户

### 功能渗透率
- **上传渗透率**：上传用户数 / DAU
- **下载渗透率**：下载用户数 / DAU
- **健康值参考**：
  - 上传渗透率 > 30%：功能健康
  - 上传渗透率 10-30%：需关注
  - 上传渗透率 < 10%：功能边缘化风险

## 异常检测规则

### 统计方法
- **2σ规则**：偏离均值2个标准差为异常
- **3σ规则**：偏离均值3个标准差为严重异常
- **趋势检测**：连续3天同向变化超过阈值

### 业务规则
- DAU单日下降 > 20%：严重异常
- MAU周环比下降 > 10%：警告
- 功能渗透率连续7天下降：关注
- 人均容量异常波动 > 50%：检查大文件影响
