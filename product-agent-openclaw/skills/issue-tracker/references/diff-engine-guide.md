# PRD 版本对比引擎指南

> 本文件定义对比新旧版 PRD 的方法、关联分析逻辑、静默修改识别策略和变更摘要输出模板，供 Issue Tracker Skill 在执行版本 Diff 时作为操作依据。

---

## 一、对比策略

### 1.1 章节级对比

PRD 文档通常具备层级标题结构。Diff 引擎按以下粒度执行对比：

**第一步：解析文档结构**

将新旧版 PRD 分别解析为章节树：

```
旧版章节树                          新版章节树
├── 1. 产品概述                      ├── 1. 产品概述
│   ├── 1.1 背景与目标               │   ├── 1.1 背景与目标  ← 内容有修改
│   └── 1.2 目标用户                 │   └── 1.2 目标用户
├── 2. 功能需求                      ├── 2. 功能需求
│   ├── 2.1 核心功能                 │   ├── 2.1 核心功能  ← 内容有修改
│   ├── 2.2 辅助功能                 │   ├── 2.2 辅助功能
│   └── 2.3 后台管理                 │   └── 2.3 数据分析  ← 新增替代原 2.3
├── 3. 非功能需求                    ├── 4. 接口定义  ← 新增一级章节
│   └── ...                          └── ...
```

**第二步：章节匹配**

按以下优先级匹配新旧章节：

1. **标题精确匹配**：章节标题完全相同时，视为同一章节
2. **标题相似匹配**：章节标题相似度 > 70%（使用编辑距离或关键词语义匹配）时，视为同一章节
3. **内容指纹匹配**：标题不同但内容相似度 > 60% 时，视为重命名
4. **无法匹配**：旧版有而新版无 → 删除；新版有而旧版无 → 新增

**第三步：内容变更检测**

对匹配上的章节对，执行内容级对比：

1. 将章节内容按段落（以空行分隔）拆分为段落列表
2. 逐段对比，标记：保持不变 / 修改 / 新增 / 删除
3. 计算内容变更比例：`修改+新增+删除的段落数 / 总段落数`

### 1.2 变更分类

| 变更类型 | 说明 | 标记 |
|---------|------|------|
| 章节新增 | 旧版中不存在的新章节 | `[新增]` |
| 章节删除 | 新版中已移除的旧章节 | `[删除]` |
| 章节重命名 | 标题变化但内容基本一致 | `[重命名]` |
| 内容修改 | 章节结构不变，内容有实质性变更 | `[修改]` |
| 内容不变 | 章节内容完全相同或仅有格式调整 | `[不变]` |
| 静默修改 | 内容变更不在原有问题清单中 | `[静默修改]` |

> 注意：格式调整（如换行、缩进、Markdown 语法变化）不计为内容修改。仅当语义内容发生变化时才标记为修改。

## 二、关联分析

### 2.1 将变更章节与问题清单关联

每个问题工单都有 `original_location` 字段，记录了该问题在旧版 PRD 中的原文定位（章节路径）。

**正向匹配**（变更覆盖了哪些问题）：

```
对于每个发生变更的章节：
  遍历工单列表：
    如果工单.original_location 匹配变更章节路径：
      将工单标记为「候选修复」
```

**匹配规则**：
- `original_location` 是章节路径时，以章节 ID 前缀匹配
- `original_location` 是原文摘录时，在新旧版本中做全文搜索定位所属章节

**反向匹配**（哪些问题未被变更覆盖）：

```
问题总数 - 正向匹配命中的问题数 = 未处理问题数
```

对未处理的问题，按以下分类标注：

| 状态 | 含义 | 行动建议 |
|------|------|---------|
| `未处理` | Diff 中未检测到相关变更 | 确认责任人和截止日期 |
| `定位失效` | 旧版章节已被删除，问题失去上下文 | 重新评估问题是否仍然适用 |
| `章节重组` | 原文可能被移动到其他章节 | 尝试在全文搜索对应内容 |

### 2.2 变更程度判定

| 变更比例 | 标签 | 说明 |
|---------|------|------|
| > 80% | 大幅重写 | 章节几乎完全重写 |
| 30% ~ 80% | 中度修改 | 章节有较多调整 |
| 5% ~ 30% | 小幅修改 | 章节有局部调整 |
| < 5% | 微调 | 几乎未变，可能未有效修复 |

> 当工单的原文定位所在章节的变更比例 < 5% 时，需在报告中标注「变更量不足，修复可能不充分」。

## 三、静默修改识别

### 3.1 什么是静默修改

**静默修改**：新版 PRD 中出现了内容变更，但变更的内容不在原有问题清单中的任何条目的原文定位范围内。

静默修改可能存在两种风险：
1. **修复引入新问题**：为修复问题 A 而改动了无关区域，引入了问题 B
2. **未申报的变更**：产品经理改了内容但没有在评审流程中申报

### 3.2 识别流程

```
对于每个「内容修改」或「章节新增」的变更：
  遍历工单列表：
    如果变更章节路径出现在任一工单的 original_location 中：
      该变更已有跟踪 ← 跳过
    如果变更章节路径未出现在任何工单中：
      该变更属于静默修改 ← 记录
```

### 3.3 静默修改的风险扫描

对识别出的静默修改内容，使用 `requirement-review/references/common-issues.md` 中的问题模式库（P-01~P-35）做快速扫描：

| 扫描步骤 | 方法 |
|---------|------|
| 搜索模糊词 | 检查是否出现 P-01~P-05 中的模糊词汇 |
| 检查结构缺失 | 检查是否缺少验收标准、异常流程、边界条件（P-06~P-10） |
| 检查一致性 | 检查新内容与已有内容的术语、数据定义是否一致（P-11~P-14） |
| 检查 NESMA 合规 | 检查是否使用了敏感词（P-18~P-24） |

扫描结果不作为「问题判定」，而是作为「风险提示」附在报告末尾，反馈给 `requirement_review` Agent 做专业评审。

### 3.4 静默修改的风险分级

| 风险等级 | 条件 | 行动 |
|---------|------|------|
| 高 | 静默修改出现在核心功能章节（P0 相关），且扫描命中致命/警告模式 | 强烈建议重新评审 |
| 中 | 静默修改出现在非核心章节，扫描命中 1-2 个建议模式 | 建议在下一轮评审中关注 |
| 低 | 静默修改内容量很小（< 3 段落），扫描未命中问题模式 | 记录备忘，不阻塞流程 |

## 四、变更摘要输出模板

Diff 分析完成后，按以下模板输出变更摘要：

```markdown
## PRD 版本变更摘要

### 基本信息
- 旧版版本号：{old_version}
- 新版版本号：{new_version}
- Diff 日期：{date}
- 总变更量：{total_changed} 处

### 章节变更明细

#### 新增章节（{n} 个）
| 章节 | 类型 | 内容量 |
|------|------|--------|
| {章节路径} | 新增 | {段落数} 段 |

#### 删除章节（{n} 个）
| 章节 | 类型 | 内容量 |
|------|------|--------|
| {章节路径} | 删除 | {段落数} 段 |

#### 修改章节（{n} 个）
| 章节 | 变更比例 | 关联问题 |
|------|---------|---------|
| {章节路径} | {x}% | IT-XXX, IT-YYY |
| {章节路径} | {x}% | 无关联问题（静默修改） |

### 问题处理覆盖

| 处理状态 | 数量 | 占比 |
|---------|------|------|
| 候选修复（Diff 有变更） | {n} | {x}% |
| 未处理（Diff 无变更） | {n} | {x}% |
| 定位失效（章节已删除） | {n} | {x}% |

### 静默修改风险提示

| 章节 | 变更量 | 风险等级 | 风险描述 |
|------|--------|---------|---------|
| {章节} | {x} 段 | 高/中/低 | {扫描命中模式说明} |

### 建议行动

1. 对 {n} 个「候选修复」工单执行复检
2. 对 {n} 个「未处理」工单确认责任人和修复计划
3. 对 {n} 个「定位失效」工单重新评估问题适用性
4. 对 {n} 处「静默修改」反馈给 requirement_review Agent 做补充评审
```

## 五、特殊场景处理

### 5.1 大规模重构

当新版 PRD 章节结构发生大规模变化（> 50% 章节无法精确匹配）时：

1. 不强行执行章节级匹配，转而使用段落级文本相似度匹配
2. 在报告中标注「PRD 经历大规模重构，章节匹配精度降低」
3. 优先通过原文摘录在新版中做全文搜索定位

### 5.2 格式转换

当 PRD 从一种格式切换到另一种格式（如从 Markdown 转为 Word 导出文本）时：

1. 先规范化格式（去除格式标记，统一换行）
2. 再执行标准对比流程
3. 格式差异不计入内容变更

### 5.3 多文件 PRD

当 PRD 拆分为多个文件时：

1. 按文件路径 + 章节路径形成完整的原文定位
2. 先做文件级匹配，再做章节级匹配
