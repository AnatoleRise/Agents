# 逻辑检测报告模板

> 本模板用于组织 PRD 逻辑检测的最终输出。设计上兼容 `requirement-review/references/report-template.md` 的格式，便于主 Agent 合并两份报告。

---

```markdown

## 基本信息
- 产品名称：{product_name}
- 文档版本：{doc_version}
- 检测模式：快速逻辑检测
- 检测日期：{date}
- 关联 Agent 评审报告：{agent_report_ref}

## 综合逻辑健康度

| 指标 | 值 |
|------|-----|
| 逻辑健康度评分 | {score}/100 |
| 等级 | {A/B/C/D} |
| 🔴致命逻辑矛盾 | {n} 个 |
| 🟡潜在逻辑风险 | {n} 个 |
| 💡逻辑优化建议 | {n} 个 |

## Top {n} 关键逻辑问题

### 1. [🔴致命] {问题标题}
- **问题模式**：{P-36 ~ P-55}
- **检测引擎**：{状态机验证/跨章节矛盾检测/数据流校验/时序依赖验证}

- **逻辑分析**：{形式化分析过程}

### 2. [🟡警告] {问题标题}
- ...

## 总结建议

{一句话总结，如：「PRD 存在 {n} 个致命逻辑矛盾，建议修复后重新检测。」}

### 7.2 综合评分的关联解读

```
逻辑健康度 = {score}/100
Agent 综合评分 = {agent_score}/100

if 逻辑健康度 >= 70 AND Agent 综合评分 >= 70:
    两份评审一致，PRD 整体质量过关
elif 逻辑健康度 >= 70 AND Agent 综合评分 < 70:
    PRD 内部逻辑自洽，但需求表达和完整性不足 → 重点修改需求表达
elif 逻辑健康度 < 70 AND Agent 综合评分 >= 70:
    PRD 写得好但逻辑有硬伤 → 重点修复业务逻辑矛盾（Agent 未发现的问题）
else:
    PRD 在多方面都有问题，建议重新梳理
```

## 与 Agent 报告合并指南

当主 Agent 需要合并逻辑检测报告与 Agent 评审报告时，按以下规则操作：

### 合并规则

1. **基本信息合并**：两份报告的"产品名称"、"文档版本"、"评审日期"保持一致
2. **评分合并**：
   - Agent 综合评分（8 维度）保留
   - 逻辑健康度评分作为独立指标追加
   - 两个评分分别保留，不做加权合并
3. **问题清单合并**：
   - Agent 的问题清单（从 P-01 到 P-35 模式）保留
   - 逻辑检测的问题清单（从 P-36 到 P-55 模式）追加
   - 两个清单合并到一个表格中，增加"来源"列标注是 Agent 还是 Skill 发现的问题
4. **交叉引用**：将「七、与 Agent 评审报告的交叉引用」章节整合到合并报告的末尾
5. **结论合并**：
   - 如果两个评估都认为 PRD 可进入开发 → 推荐进入开发
   - 如果任一评估发现致命问题 → 不推荐进入开发，注明问题来源
