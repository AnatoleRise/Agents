# PRD 评审报告模板

## 快速扫描报告模板

```markdown
# PRD 快速评审报告

## 基本信息
- 产品名称：{product_name}
- 文档版本：{doc_version}
- 评审模式：快速扫描
- 评审日期：{date}
- 版本号：v{version}
- 激活角色：{role1, role2, ...}

## 总体评分

| 指标 | 值 |
|------|-----|
| 综合评分 | {score}/100 |
| 评级 | {A/B/C/D} |
| 🔴严重 问题 | {n} 个 |
| 🟡警告 问题 | {n} 个 |
| 💡建议 问题 | {n} 个 |

## Top {n} 关键问题

### 1. [🔴严重] {问题标题}
- **角色视角**：{测试/前端/后端/...}
- **位置**：{章节名}
- **描述**：{问题描述}
- **影响**：{影响说明}
- **建议**：{修改建议}

### 2. [🟡警告] {问题标题}
- ...

## 总结建议

{一句话总结，如：「当前 PRD 存在 {n} 个严重问题需要修复，建议在解决严重项后重新评审。」}
```

## 深度评审报告模板（含多角色评审）

```markdown
# PRD 深度评审报告

## 基本信息
- 产品名称：{product_name}
- 文档版本：{doc_version}
- 评审模式：深度评审
- 评审日期：{date}
- 版本号：v{version}
- 评审人：PRD Reviewer Agent

## 一、总体评估

| 指标 | 值 |
|------|-----|
| 综合评分 | {score}/100 |
| 评级 | {A/B/C/D} |
| 🔴严重 问题 | {n} 个 |
| 🟡警告 问题 | {n} 个 |
| 💡建议 问题 | {n} 个 |
| 建议 | {可直接进入开发 / 修改后重新评审 / 需重大修改} |

## 二、评审报告（通用 8 大维度）

### 2.1 维度评分矩阵

| 维度 | 得分 | 问题数 | 权重 |
|------|------|--------|------|
| 完整性 | {score}/100 | {n} | 20% |
| 一致性 | {score}/100 | {n} | 15% |
| 可测试性 | {score}/100 | {n} | 20% |
| 可行性 | {score}/100 | {n} | 10% |
| 清晰度 | {score}/100 | {n} | 15% |
| 优先级 | {score}/100 | {n} | 5% |
| 边界场景 | {score}/100 | {n} | 10% |
| 合规性 | {score}/100 | {n} | 5% |

**加权总分** = Σ(维度得分 × 权重) = {final_score}/100

### 2.2 问题清单（按严重程度）

#### 🔴严重 问题

**S-1: {问题标题}**
- **所在维度**：{完整性/一致性/...}
- **所在章节**：{章节路径}
- **原文引用**：> {原文内容}
- **问题描述**：{详细描述}
- **影响分析**：{对开发/测试/上线的影响}
- **修改建议**：{具体建议}
- **改写示例**：
  ```
  原文：{原文}
  建议改为：{改写后}
  ```

#### 🟡警告 问题

**J-1: {问题标题}**
- **所在维度**：{维度名}
- ...

#### 💡建议 问题

**JY-1: {问题标题}**
- **所在维度**：{维度名}
- ...

### 2.3 各维度详细发现

#### 2.3.1 完整性检查
- **得分**：{score}/100
- **缺失项**：{列出缺失的必要章节或信息}
- **亮点**：{做得好的地方}

#### 2.3.2-2.3.8（其他维度同上格式）

## 三、角色评审概览

| 角色 | 激活状态 | 问题数 | 关键发现 |
|------|---------|--------|---------|
| 专家产品经理 | ✅ 已激活 | {n} | {一句话总结} |

| UI/UX 设计师 | ✅ 已激活 | {n} | {一句话总结} |
| 前端工程师 | ✅ 已激活 | {n} | {一句话总结} |
| 后端工程师 | ✅ 已激活 | {n} | {一句话总结} |
| 算法工程师 | ✅ 已激活 | {n} | {一句话总结} |
| 数据工程师 | ❌ 未激活 | - | PRD 未涉及埋点/数据内容 |
```
